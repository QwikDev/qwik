Create `src/optimizer/context-stack.ts` implementing a `ContextStack` class that manages the naming hierarchy during AST traversal.

The class wraps an internal string array and provides push/pop operations for AST node types. It also integrates with Phase 1 naming utilities to produce displayName and symbolName on demand.

```typescript
export class ContextStack {
  private stack: string[] = [];
  private fileStem: string;
  private relPath: string;
  private scope: string | undefined;

  constructor(fileStem: string, relPath: string, scope?: string);

  // Push context for naming based on AST node type
  push(name: string): void;
  pop(): void;

  // Get current context for naming
  getDisplayName(): string;    // calls buildDisplayName(fileStem, stack)
  getSymbolName(): string;     // calls buildSymbolName(displayName, scope, relPath)
  getContextStack(): string[]; // returns copy of current stack

  // Current depth (for nesting detection)
  get depth(): number;
}
```

**Stack push rules** (verified from snapshot analysis):
- VariableDeclarator with Identifier id: push `node.id.name`
- FunctionDeclaration with id: push `node.id.name`
- Property/MethodDefinition key (Identifier): push `key.name`
- JSXOpeningElement tag (JSXIdentifier): push `tagName`
- JSXAttribute with `$`-suffixed name: push the attribute name (e.g., `"onClick$"`)
- ExportDefaultDeclaration without named declaration: push file stem derived from relPath
  - For bracket-style routes like `[[...slug]].tsx`, extract just `slug` (strip brackets, dots, extension)

**File stem extraction for default exports:**
- `test.tsx` -> `test`
- `index.tsx` -> `index`
- `[[...slug]].tsx` -> `slug` (strip leading `[[...` and trailing `]]`)
- `[id].tsx` -> `id` (strip brackets)

The constructor receives `fileStem` (already the display-name prefix like `"test.tsx"`) and `relPath` for hashing.

Note: The ContextStack does NOT do the AST walking itself. It is a data structure that the walker (in extract.ts, Plan 03) calls push/pop on. This plan only tests the stack logic and naming integration.

**Key types and exports:**
```typescript
export interface ImportInfo {
  localName: string;       // The local binding name
  importedName: string;    // The original exported name
  source: string;          // The module source (e.g., "@qwik.dev/core")
  isQwikCore: boolean;     // Whether source is @qwik.dev/core or @builder.io/qwik
}

export interface CustomInlinedInfo {
  dollarName: string;   // e.g., "useMemo$"
  qrlName: string;      // e.g., "useMemoQrl"
}

// Collect all import declarations from a parsed AST Program node
export function collectImports(program: any): Map<string, ImportInfo>;

// Scan module body for `export const X$ = wrap(XQrl)` patterns
export function collectCustomInlined(program: any): Map<string, CustomInlinedInfo>;

// Get the callee name from a CallExpression node (handles Identifier only for now)
export function getCalleeName(callExpr: any): string | null;

// Check if a CallExpression is a marker call that should trigger extraction
export function isMarkerCall(
  callExpr: any,
  imports: Map<string, ImportInfo>,
  customInlined: Map<string, CustomInlinedInfo>
): boolean;

// Check if callee is bare "$" vs "component$" etc.
export function isBare$(callExpr: any): boolean;

// Determine ctxKind for the marker: "function" or "eventHandler"
// eventHandler is for JSX event attributes (onClick$, onChange$, etc.)
export function getCtxKind(calleeName: string, isJsxEventAttr: boolean): 'function' | 'eventHandler';

// Determine ctxName: the marker function name itself (e.g., "$", "component$", "useTask$", "onClick$")
export function getCtxName(calleeName: string, isJsxEventAttr: boolean, jsxAttrName?: string): string;
```

**isMarkerCall logic:**
1. Get callee name via getCalleeName()
2. If name does not end with `$`, return false
3. If name is in the imports map AND source isQwikCore, return true (built-in marker)
4. If name is in the customInlined map, return true (user-defined marker)
5. If name is `$` (bare dollar) and `$` is in imports from qwik core, return true
6. Otherwise return false

**collectImports logic:**
For each ImportDeclaration in program.body:
- Check if source.value is `@qwik.dev/core`, `@builder.io/qwik`, or their sub-paths
- For each specifier, create ImportInfo with localName, importedName, source
- Mark isQwikCore = true if source starts with `@qwik.dev/core` or `@builder.io/qwik`

**collectCustomInlined logic:**
For each ExportNamedDeclaration with VariableDeclaration:
- If pattern matches `export const X$ = wrap(XQrl)` or `export const X$ = /*@__PURE__*/ wrap(XQrl)`
- Where X$ ends with `$` and the wrap argument ends with `Qrl`
- Register in map: `X$` -> `{ dollarName: X$, qrlName: XQrl }`

**sync$ detection:** `sync$` is a special case -- it IS a marker but does NOT extract a segment. Add:
```typescript
export function isSyncMarker(calleeName: string): boolean {
  return calleeName === 'sync$';
}
```
