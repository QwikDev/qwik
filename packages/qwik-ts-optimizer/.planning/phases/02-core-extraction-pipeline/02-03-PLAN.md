Create `src/optimizer/extract.ts` -- the core extraction engine. This module:

1. Parses source with oxc-parser
2. Collects imports and custom inlined functions
3. Walks the AST with oxc-walker, maintaining ContextStack
4. When a marker call is found: extracts segment info (body text, position, metadata)
5. Returns an array of ExtractionResult objects

**ExtractionResult type:**
```typescript
export interface ExtractionResult {
  // Identity
  symbolName: string;         // e.g., "Header_component_render_onClick_jMxQsjbyDss"
  displayName: string;        // e.g., "test.tsx_Header_component_render_onClick"
  hash: string;               // 11-char base64 hash
  canonicalFilename: string;  // symbolName used in import path

  // Positions in original source (for magic-string)
  callStart: number;          // Start of the entire CallExpression
  callEnd: number;            // End of the entire CallExpression
  calleeStart: number;        // Start of callee identifier
  calleeEnd: number;          // End of callee identifier
  argStart: number;           // Start of first argument (the closure)
  argEnd: number;             // End of first argument

  // Segment content
  bodyText: string;           // Original source text of the extracted argument

  // Call form info
  calleeName: string;         // e.g., "component$", "$", "sync$"
  isBare: boolean;            // true if callee is just "$"
  isSync: boolean;            // true if callee is "sync$"
  qrlCallee: string;          // e.g., "componentQrl", "" for bare $

  // Metadata
  ctxKind: 'function' | 'eventHandler';
  ctxName: string;
  origin: string;             // relPath
  extension: string;          // ".tsx", ".ts", or ".js"
  loc: [number, number];      // [startLine, startCol] of the closure argument
  parent: string | null;      // parent segment symbolName (null for top-level)
  captures: boolean;          // Phase 2: set to false, Phase 3 will compute

  // Imports needed by this segment
  segmentImports: ImportInfo[];  // Imports referenced within the segment body
}
```

**extractSegments function:**
```typescript
export function extractSegments(
  source: string,
  relPath: string,
  scope?: string
): ExtractionResult[];
```

**Algorithm:**
1. Parse source with `parseSync(relPath, source)` from oxc-parser
2. `collectImports(program)` and `collectCustomInlined(program)`
3. Derive fileStem from relPath (basename without extension, but the full basename like "test.tsx" for display name prefix)
4. Create `ContextStack(fileStem, relPath, scope)`
5. Walk with oxc-walker, pushing/popping context for naming:
   - On enter VariableDeclarator with Identifier id: push name
   - On enter FunctionDeclaration with id: push name
   - On enter Property/MethodDefinition with Identifier key: push key.name
   - On enter JSXOpeningElement: push tag name
   - On enter JSXAttribute with `$`-suffixed name: push attr name
   - On enter ExportDefaultDeclaration: push file stem if no named child
   - On enter CallExpression that isMarkerCall: record extraction
   - On leave: pop matching contexts
6. For each marker call found:
   - Extract bodyText from source using `source.slice(arg.start, arg.end)`
   - Compute displayName and symbolName from ContextStack
   - Determine extension by checking if bodyText contains JSX (`<` followed by an identifier -- or more precisely, check if the AST subtree of the argument contains JSXElement/JSXFragment nodes)
   - Determine qrlCallee: if bare $, qrlCallee = ""; otherwise strip trailing `$` and append `Qrl`
   - Set captures = false (Phase 3 concern)
   - Set parent = null for now (Plan 04 handles nesting)

**Segment import detection (IMP-05):**
For each extraction, determine which imports the segment body references. Walk the segment body AST subtree and collect all Identifier references. Cross-reference against the imports map. Only include imports where the segment body uses the local name. This is a simplified check -- full accuracy comes from scope analysis in Phase 3.

**Extension determination (Pattern 8 from research):**
```typescript
function determineExtension(argNode: any, sourceExt: string): string {
  // Check if the argument AST subtree contains JSXElement or JSXFragment
  let hasJsx = false;
  walk(argNode, {
    enter(node) {
      if (node.type === 'JSXElement' || node.type === 'JSXFragment') {
        hasJsx = true;
      }
    }
  });
  if (hasJsx) return '.tsx';
  if (sourceExt === '.ts') return '.ts';
  return '.js';
}
```

```typescript
// Get the Qrl callee name from a marker name
// "component$" -> "componentQrl", "$" -> "", "sync$" -> "_qrlSync"
export function getQrlCalleeName(markerName: string): string;

// Build a QRL const declaration string
// Returns: `const q_{symbolName} = /*#__PURE__*/ qrl(()=>import("./{canonicalFilename}"), "{symbolName}");`

  canonicalFilename: string
): string;

// Build the sync$ transformation
// Returns: `_qrlSync({originalFnText}, "{minifiedFnText}")`
export function buildSyncTransform(originalFnText: string): string;

// Determine if a QRL wrapper call needs /*#__PURE__*/ annotation
// true for: componentQrl, qrl (in declarations)
// false for: useTaskQrl, useStylesQrl, etc.
export function needsPureAnnotation(qrlCalleeName: string): boolean;

// Get the import source for a Qrl callee
// Most come from @qwik.dev/core, but qwikifyQrl from @qwik.dev/react
export function getQrlImportSource(qrlCalleeName: string): string;
```

**getQrlCalleeName logic:**
- If markerName is `"$"`: return `""` (bare QRL, no wrapper)
- If markerName is `"sync$"`: return `"_qrlSync"`
- Otherwise: strip trailing `$`, append `Qrl` (e.g., `"component$"` -> `"componentQrl"`)

**buildQrlDeclaration format (VERIFIED from all snapshots):**
```
const q_{symbolName} = /*#__PURE__*/ qrl(()=>import("./{canonicalFilename}"), "{symbolName}");
```
Note the exact spacing: no space after `=>`, one space after `/*#__PURE__*/`.

**buildSyncTransform (Pattern 7):**
- Takes the original function text (source.slice(arg.start, arg.end))
- Produces a minified version of the function for the string argument
- Minification: strip comments, collapse whitespace to single spaces, strip unnecessary whitespace around operators/braces/parens
- Return: `_qrlSync({originalFnText}, "{minifiedText}")`
- For the minified text, a simple approach: remove `//` line comments and `/* */` block comments, collapse consecutive whitespace to single space, trim. This doesn't need to be perfect -- it needs to match SWC's serialization.

**needsPureAnnotation logic (VERIFIED from snapshots):**
- `"componentQrl"` -> true
- `"qrl"` -> true (used in QRL const declarations)
- All others (`"useTaskQrl"`, `"useStylesQrl"`, `"serverQrl"`, etc.) -> false

**getQrlImportSource logic:**
- `"qwikifyQrl"` -> `"@qwik.dev/react"`
- Everything else -> `"@qwik.dev/core"`
