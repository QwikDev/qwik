**Main export:**
```typescript
export interface ParentRewriteResult {
  code: string;                     // Rewritten parent module source
  extractions: ExtractionResult[];  // All extractions (possibly with nested parent refs)
}

export function rewriteParentModule(
  source: string,
  relPath: string,
  extractions: ExtractionResult[],
  originalImports: Map<string, ImportInfo>
): ParentRewriteResult;
```

**Algorithm (using magic-string):**

1. Create `new MagicString(source)`

2. **Rewrite existing import sources** (IMP-01..03):
   For each ImportDeclaration in the original AST:
   - Find the string literal source node
   - If `rewriteImportSource(source.value)` differs from original, use `s.overwrite(sourceNode.start + 1, sourceNode.end - 1, newSource)` (overwrite just inside the quotes)

3. **Rewrite call sites** in the source:
   For each extraction (sorted by callStart ascending):
   - If `isSync`: replace entire call with `buildSyncTransform(bodyText)`. Add `_qrlSync` to needed imports.
   - If `isBare` ($): replace entire CallExpression with `q_${symbolName}`:
     `s.overwrite(ext.callStart, ext.callEnd, 'q_' + ext.symbolName)`
   - Else (component$, useTask$, etc.):
     - Replace callee: `s.overwrite(ext.calleeStart, ext.calleeEnd, ext.qrlCallee)`
     - Replace argument: `s.overwrite(ext.argStart, ext.argEnd, 'q_' + ext.symbolName)`
     - If `needsPureAnnotation(ext.qrlCallee)`: prepend `/*#__PURE__*/ ` before the callee
       Use `s.appendLeft(ext.callStart, '/*#__PURE__*/ ')` — but only for componentQrl calls. Note: the PURE annotation on QRL declarations is handled by buildQrlDeclaration, not here.

4. **Remove $-function import specifiers** from existing import statements:
   For marker imports (component$, useTask$, $, etc.) that were imported from qwik core:
   - These are no longer needed in their $ form
   - Remove the import specifier. If it was the only specifier, remove the entire import statement.
   - Be careful: some imports have both marker and non-marker specifiers (e.g., `import { component$, useStore } from "@qwik.dev/core"` -- remove `component$` but keep `useStore`)

5. **Build optimizer-added imports** (IMP-04):
   Collect all unique imports needed:
   - `qrl` from `@qwik.dev/core` (always needed if any segments extracted)
   - Each unique `qrlCallee` (e.g., `componentQrl`, `useTaskQrl`) from its source
   - `_qrlSync` from `@qwik.dev/core` if any sync$ calls
   - IMPORTANT: Each import is a SEPARATE statement: `import { qrl } from "@qwik.dev/core";`
   - IMPORTANT: Do NOT add an import if the symbol is already imported (IMP-06 dedup)
   - Sort these import statements alphabetically by the imported symbol name

6. **Build QRL declarations:**
   For each non-sync extraction, generate: `buildQrlDeclaration(ext.symbolName, ext.canonicalFilename)`
   Sort alphabetically by symbolName.

7. **Assemble final output:**
   Prepend to the (already-modified) source:
   ```
   {optimizer-added imports}\n//\n{QRL declarations}\n//\n
   ```
   Use `s.prepend(...)` to add this before the modified source body.

8. Return `{ code: s.toString(), extractions }`

**Nested segment handling (EXTRACT-03):**
When extractSegments finds a marker call whose parent in the AST is inside another marker call's argument, the inner segment gets `parent: outerSegment.symbolName`. The extraction engine (Plan 03) sets this up. In rewriteParentModule:
- Inner segments also get QRL declarations in the parent module
- Inner segments also get their own segment module code (handled by segment-codegen)
- The parent field in SegmentAnalysis is populated from ExtractionResult.parent

For nested segments within a segment's body: the segment's own module will have QRL declarations for its children. This means `generateSegmentCode()` (from Plan 03) needs to receive the nested QRL declarations. The rewriteParentModule function should separate top-level extractions (parent=null) from nested ones and include nested QRL declarations in the correct scope.

**Custom inlined function handling (EXTRACT-06):**
Custom inlined functions (detected by collectCustomInlined in Plan 02) are handled the same way as built-in markers:
- `useMemo$(() => ...)` -> extract segment, rewrite to `useMemoQrl(q_symbolName)`
- The `qrlCallee` comes from the CustomInlinedInfo.qrlName
- Import for `useMemoQrl` is NOT added (it's defined locally in the same file)
