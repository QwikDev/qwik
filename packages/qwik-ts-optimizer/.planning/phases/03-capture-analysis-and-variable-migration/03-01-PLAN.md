Create `src/optimizer/capture-analysis.ts` with:

1. Export `CaptureAnalysisResult` interface:
   ```typescript
   export interface CaptureAnalysisResult {
     captureNames: string[];   // alphabetically sorted captured variable names
     captures: boolean;        // true if captureNames.length > 0
     paramNames: string[];     // function formal parameter names
   }
   ```

2. Export `analyzeCaptures()` function:
   ```typescript
   export function analyzeCaptures(
     closureNode: any,           // ArrowFunctionExpression or FunctionExpression (the $() argument)
     parentScopeIdentifiers: Set<string>,  // identifiers declared in enclosing scope(s)
     importedNames: Set<string>,           // names from import statements
   ): CaptureAnalysisResult
   ```

   Implementation:
   a. Extract paramNames from `closureNode.params` — walk each param to collect all Identifier binding names (handles simple identifiers, destructured objects `{a, b}`, destructured arrays `[a, b]`, rest patterns `...rest`, nested patterns, defaults). Store as string array.
   b. Call `getUndeclaredIdentifiersInFunction(closureNode)` from oxc-walker to get all identifiers referenced but not declared within the closure.
   c. Filter the undeclared list to ONLY include names that exist in `parentScopeIdentifiers` AND are NOT in `importedNames`.
   d. This filtering excludes: globals (console, Math, JSON, setTimeout, etc.), import bindings, and the function's own parameters (which getUndeclaredIdentifiersInFunction already excludes since they ARE declared within the function).
   e. Sort filtered captures alphabetically.
   f. Return `{ captureNames: sorted, captures: sorted.length > 0, paramNames }`.

3. Export `collectParamNames()` helper:
   ```typescript
   export function collectParamNames(params: any[]): string[]
   ```
   Walks parameter AST nodes to extract all binding names. Handles:
   - Simple `Identifier` → `[name]`
   - `ObjectPattern` with properties → recursively collect each property value's bindings
   - `ArrayPattern` → recursively collect each element's bindings
   - `RestElement` → recursively collect argument's bindings
   - `AssignmentPattern` (defaults) → collect from left side

4. Export `collectScopeIdentifiers()` helper — given an AST node (function body or program), use oxc-walker's ScopeTracker to collect all identifiers declared in that scope (not nested scopes). This provides the `parentScopeIdentifiers` set.
   ```typescript
   export function collectScopeIdentifiers(
     containerNode: any,
     source: string,
     relPath: string,
   ): Set<string>
   ```
   Implementation: Parse with oxc-parser, walk with ScopeTracker, collect all declarations at the scope level of the container. For function scope containers, include var-hoisted declarations from nested blocks.
