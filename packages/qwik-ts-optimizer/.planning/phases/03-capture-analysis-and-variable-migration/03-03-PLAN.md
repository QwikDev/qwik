export interface SegmentCaptureInfo {
  captureNames: string[];         // variables received via _captures
  autoImports: Array<{ varName: string; parentModulePath: string }>;  // _auto_VARNAME imports
  movedDeclarations: string[];    // declaration text physically moved into segment

Update `generateSegmentCode(extraction, nestedQrlDecls?, captureInfo?)`:

**When captureInfo.captureNames has entries (scope-level captures):**
1. Add `import { _captures } from "@qwik.dev/core";` to the imports section. Group it with other @qwik.dev/core imports if any exist, or as a separate import statement.
2. After the export const line's opening `{` (or for arrow expressions, convert to block body), prepend the unpacking:
   ```
   const var1 = _captures[0], var2 = _captures[1];
   ```
   The format is a SINGLE `const` declaration with comma-separated declarators, semicolon at end. Index order matches alphabetical captureNames order.
3. The _captures unpacking goes INSIDE the function body, right after the opening brace.

Implementation approach for _captures injection into bodyText:
- The bodyText is the raw source text of the closure (e.g., `() => { ... }` or `(params) => expr`)
- Find the opening `{` of the function body. For block bodies: after `=> {` or `() {`. For expression bodies (`=> expr`): convert to `=> { return expr; }` first.
- Insert the unpacking line right after `{` with a newline.
- The resulting bodyText is used in the `export const symbolName = <modified bodyText>;` output.

**When captureInfo.autoImports has entries (module-level migration):**
1. For each autoImport, add: `import { _auto_VARNAME as VARNAME } from "parentModulePath";`
   The parentModulePath has NO extension (e.g., `"./test"` not `"./test.tsx"`).
2. These go in the imports section alongside other segment imports.

**When captureInfo.movedDeclarations has entries:**
1. Insert the moved declaration text BEFORE the `export const` line, after any `//` separator.
2. Each moved declaration is a complete statement (e.g., `const helperFn = (msg) => console.log(msg);`).

Output structure becomes:
```
[existing imports]
[_captures import if needed]
[_auto_ imports if needed]
//
[moved declarations if any]
[nested QRL declarations if any]
//
export const symbolName = bodyText;  (with _captures unpacking injected if needed)
```

Add two new steps:

**Step 4b: .w() wrapping (CAPT-03):**
After Step 4 (rewrite call sites), for each top-level extraction that has `captureNames.length > 0`:
- Find the QRL reference text that was written (e.g., `q_symbolName`)
- Append `.w([\n        var1,\n        var2\n    ])` to it
- Use `s.appendRight()` at the position right after the QRL reference text end
- Variables in .w() are in the SAME alphabetical order as captureNames

Implementation detail: After Step 4 rewrites `ext.argStart..ext.argEnd` to `q_symbolName`, the MagicString position for the end of `q_symbolName` is `ext.argEnd`. Use `s.appendLeft(ext.argEnd, '.w([\n        ' + ext.captureNames.join(',\n        ') + '\n    ])')` to append after the QRL ref.

For bare $() extractions: the entire call was replaced with `q_symbolName`. Append .w() at `ext.callEnd`.

For named markers (component$, useTask$, etc.): the arg was replaced with `q_symbolName`. Append .w() at `ext.argEnd`.

**Step 6b: _auto_ exports:**
If `migrationDecisions` is provided, for each decision with `action === 'reexport'`:
- Append `export { VARNAME as _auto_VARNAME };` at the END of the module
- Use `s.append('\nexport { ' + decision.varName + ' as _auto_' + decision.varName + ' };')` after all other code

**Step 6c: Remove migrated declarations:**
