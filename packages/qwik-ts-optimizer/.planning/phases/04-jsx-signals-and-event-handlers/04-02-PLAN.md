    Create `src/optimizer/signal-analysis.ts` with:

    1. `SignalExprResult` type:
       ```typescript
       type SignalExprResult =
         | { type: 'none' }
         | { type: 'wrapProp'; code: string }
         | { type: 'fnSignal'; deps: string[]; hoistedFn: string; hoistedStr: string }
       ```

    2. `analyzeSignalExpression(exprNode: any, source: string, importedNames: Set<string>): SignalExprResult`

       Detection rules (from RESEARCH.md Pattern 6):

       **_wrapProp(signal)** — when the entire expression is a MemberExpression accessing `.value`:
       - `node.type === 'MemberExpression'` AND `node.property.name === 'value'` (or `node.computed === false && property is Identifier 'value'`)
       - AND the parent is NOT a CallExpression (i.e., `signal.value()` must NOT be wrapped)
       - Output: `_wrapProp(objectSourceText)`

       **_wrapProp(store, "field")** — when the entire expression is a MemberExpression accessing a named field on a non-signal object:
       - `node.type === 'MemberExpression'` AND property is NOT `.value`
       - AND the object is a known store/props reference (heuristic: object is an identifier not in importedNames and not a well-known global)
       - Computed access (`props['data-nu']`): `_wrapProp(obj, "data-nu")`
       - Dot access (`props.class`): `_wrapProp(obj, "class")`

       **NOT wrapped (SIG-05)**:
       - `signal.value()` — CallExpression where callee is `signal.value`
       - `signal.value + unknown()` — BinaryExpression containing CallExpression with non-imported callee
       - `mutable(signal)` — CallExpression with callee `mutable`
       - `signal.value + dep` — BinaryExpression mixing signal.value with import reference
       - Any expression that's just a literal, import, or non-signal identifier

    3. `isSignalValueAccess(node: any): boolean` — helper to detect `x.value` MemberExpression pattern.

    4. `isStoreFieldAccess(node: any, importedNames: Set<string>): boolean` — helper to detect `props.field` or `store.field` patterns where the object is a local variable (not imported).

    NOTE: The `_wrapProp` for store fields applies specifically to props and stores — objects that are reactive containers. The object must NOT be in importedNames (imports are static).
