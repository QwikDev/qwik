  <action>
    Create `src/optimizer/event-handler-transform.ts` with:

    1. `isEventProp(propName: string): boolean` — returns true if prop starts with `on` (or has scope prefix `document:on`, `window:on`) and ends with `$`. Does NOT match non-event `$` props like `transparent$`.

    2. `isPassiveDirective(propName: string): boolean` — returns true for `passive:*` props.

    3. `transformEventPropName(propName: string, passiveEvents: Set<string>): string | null`

       Naming rules (from RESEARCH.md Pattern 5):
       a. Strip the `$` suffix
       b. Check for scope prefix:
          - `document:onXxx` -> prefix = `q-d:`, strip `document:on` prefix
          - `window:onXxx` -> prefix = `q-w:`, strip `window:on` prefix
          - `host:onXxx` -> return prop as-is (passthrough, no transformation)
          - No prefix, starts with `on` -> prefix = `q-e:`, strip `on` prefix
       c. Extract event name from remaining string:
          - If starts with `-` (like `on-customName`): strip leading `-`, convert camelCase to kebab-case
          - Otherwise: convert entire name to lowercase
       d. Check passive: if the extracted event name is in passiveEvents set:
          - `q-e:` -> `q-ep:`
          - `q-d:` -> `q-dp:`
          - `q-w:` -> `q-wp:`
       e. Return `"prefix:eventName"` as a quoted string prop key

    4. `camelToKebab(str: string): string` — convert camelCase to kebab-case. `anotherCustom` -> `another-custom`, `DblClick` -> `dblclick`.

    5. `collectPassiveDirectives(attributes: any[]): Set<string>` — scan JSX attributes for `passive:*` props, extract event names, return set. Passive-only attributes (no matching handler) should be stripped.

    6. `EventTransformResult` type:
       ```typescript
       interface EventTransformResult {
         outputPropName: string;  // e.g., "q-e:click"
         isHtmlOnly: boolean;     // true = only transform on HTML elements
         stripPassive: string[];  // passive:* props to remove
       }
       ```

    NOTE: The actual segment extraction of event handler closures (EVT-06) is already handled by the existing extraction engine in extract.ts — it detects `$`-suffixed calls within JSX attributes. This module only handles the prop NAMING transformation, not extraction.
  </action>
