    ```typescript
    import { analyzeSignalExpression, SignalHoister, type SignalExprResult } from './signal-analysis.js';
    ```

    **2. Modify processProps() signature and body:**

    Add parameters: `tagIsHtml: boolean`, `passiveEvents: Set<string>`, `signalHoister: SignalHoister`.
    Add to return type: `neededImports: Set<string>` and `hoistedSignalDecls: string[]`.

    In the per-attribute loop (after extracting propName and valueText/valueNode), add this processing order BEFORE the existing classifyProp call:

    a. **Strip passive: directives** — if `isPassiveDirective(propName)`, skip the attribute entirely (passive directives are consumed by collectPassiveDirectives and should not appear in output).

    b. **Bind desugaring** — if `isBindProp(propName)` AND this is NOT a spread element:
       - Call `transformBindProp(propName, valueText)` to get the result
       - Add the value prop: `constEntries.push('"' + result.propName + '": ' + result.propValue)`
       - If result.handler is not null, track the handler entry for `"q-e:input"` (to be merged later if another handler targets the same event)
       - Add result.needsImport items to a local neededImports set
       - `continue` to next attribute (don't run classify/signal/event logic on bind props)

    c. **Event prop renaming** — if `isEventProp(propName)` AND `tagIsHtml`:
       - Call `transformEventPropName(propName, passiveEvents)` to get the renamed prop name
       - If result is not null, use the renamed name (e.g., `"q-e:click"`) instead of the original `onClick$`
       - The VALUE stays the same (it's the QRL reference injected by parent rewriting)
       - Event handler props always go to constEntries (per RESEARCH Pattern 2)
       - `continue` to next attribute after pushing to constEntries

    d. **Signal analysis** — for all other props with a valueNode:
       - Call `analyzeSignalExpression(valueNode, source, importedNames)`
       - If result.type === 'wrapProp': replace valueText with result.code, push to constEntries (signal-wrapped props are always const per RESEARCH Pattern 2), add '_wrapProp' to neededImports
       - If result.type === 'fnSignal': call `signalHoister.hoist(result.hoistedFn, result.hoistedStr)` to get the _hfN name, build the _fnSignal call: `_fnSignal(${hfName}, [${result.deps.join(', ')}], ${hfName}_str)`, push to constEntries, add '_fnSignal' to neededImports
       - If result.type === 'none': fall through to existing classifyProp logic

    e. **Existing classifyProp fallback** — for props where signal analysis returned 'none':
       - Keep the existing classifyProp(valueNode, importedNames) logic unchanged
       - Push to varEntries or constEntries as before

    **After the loop**, handle bind handler merging:
    - If any bind handlers were collected for "q-e:input", check if there's also an explicit onInput$ handler in the constEntries
    - If both exist, merge using mergeEventHandlers() and replace the entry
    - If only bind handler, add it to constEntries

    **3. Update transformJsxElement():**
    - Pass `tagIsHtml` (from `isHtmlElement(tag)` check — but note tag is the processed string like `"div"` so check the raw tag name), `passiveEvents`, and `signalHoister` to processProps()
    - Collect neededImports from processProps() result and merge into the element's neededImports set

    **4. Update transformAllJsx():**
    - Create a `SignalHoister` instance at the start
    - Pass it through to transformJsxElement (which passes to processProps)
    - After the walk completes, get hoisted declarations via `signalHoister.getDeclarations()`
    - Update `JsxTransformOutput` to include `hoistedDeclarations: string[]`
    - For each JSXElement, call `collectPassiveDirectives(openingElement.attributes)` and pass the result to processProps

    **5. Update rewrite-parent.ts to use hoisted declarations:**
    - After the JSX transform call at line ~307, if `jsxResult.hoistedDeclarations.length > 0`, prepend them to the source using `s.prepend()` or insert after imports
    - These are the `const _hf0 = ...; const _hf0_str = ...;` lines

    **Key correctness rules from RESEARCH.md:**
    - Event handler q-e: renaming ONLY on HTML elements (lowercase tag), NOT on components (Pattern 10)
    - Signal-wrapped props (_wrapProp, _fnSignal) always go to constProps (Pattern 2)
    - bind: with spread goes to varProps unchanged (Pattern 8 "bind with spread")
    - passive: directives are stripped from output (Pattern 5)
    - Event handlers from $-suffixed props on components keep their original name
