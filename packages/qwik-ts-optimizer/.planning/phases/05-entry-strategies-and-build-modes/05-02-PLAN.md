Wire inline/hoist strategy into the pipeline:
1. In transform.ts, detect if entryStrategy is 'inline' or 'hoist'
2. If inline/hoist: do NOT emit separate segment TransformModules. Instead, all segment code is inlined into the parent.
3. In rewrite-parent.ts: add an inline strategy branch. When inline/hoist:
   a. Replace QRL declarations (Step 5) with _noopQrl declarations using buildNoopQrlDeclaration (or buildNoopQrlDevDeclaration in dev mode, or buildStrippedNoopQrl for stripped segments)
   b. Track a stripped segment counter for sentinel naming
   c. After QRL declarations and before exports: emit .s() calls that attach segment bodies. Order: stripped/noop segments first, then non-stripped, then component body last.
   d. Import _noopQrl (or _noopQrlDEV) from @qwik.dev/core instead of qrl

   f. _captures import still added inside .s() bodies that need it
