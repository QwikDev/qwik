       - `test_input!` macro calls with options like `transpile_ts`, `transpile_jsx`, `entry_strategy`, `explicit_extensions`, `is_server`, `strip_exports`, `strip_ctx_name`, `reg_ctx_name`, `mode`, `preserve_filenames`, `scope`, `minify`

  <action>
    1. **Import cleanup** in `src/optimizer/rewrite-parent.ts` (or as a post-processing step in transform.ts):
       After parent module rewriting is complete (extractions moved to segments, QRL declarations added), scan the final parent module code to find non-Qwik import specifiers whose imported names no longer appear in the parent body. Remove those import declarations/specifiers using magic-string.

       Implementation approach:
       - Re-parse the rewritten parent code with oxc-parser
       - Collect all ImportDeclaration nodes and their imported specifier names
       - Scan the rest of the AST (excluding import declarations) for Identifier references
       - For each import specifier: if its local name is NOT referenced anywhere in the parent body, mark for removal
       - Do NOT remove Qwik imports (they may have been added by the rewriter)
       - Do NOT remove side-effect imports (`import 'module'` with no specifiers)
       - Use magic-string to remove unused import specifiers or entire import declarations
       - This is NOT DCE -- it only cleans up imports after extraction moved their consumers to segments

    2. **explicitExtensions** threading:
       - In `src/optimizer/rewrite-calls.ts`: modify `buildQrlDeclaration()` to accept an optional `explicitExtensions: boolean` parameter. When true, append `.js` to the import path in the QRL declaration.
       - In `src/optimizer/transform.ts`: thread `options.explicitExtensions` through to `buildQrlDeclaration()` calls (both for top-level and nested QRL declarations).
       - In `src/optimizer/segment-codegen.ts`: if segment codegen produces QRL import references for nested segments, also append `.js` when explicitExtensions is true.

    3. Update `tests/optimizer/snapshot-batch.test.ts` if needed to pass explicitExtensions for relevant snapshots.

    4. Run the convergence test to measure improvement: the import cleanup and explicitExtensions fixes should increase the pass rate from 2/209 significantly.
