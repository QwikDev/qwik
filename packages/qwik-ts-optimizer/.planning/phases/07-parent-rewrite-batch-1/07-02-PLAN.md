Add a `transformSCallBody()` function in `rewrite-parent.ts` that transforms an extraction's body text for use in `.s()` calls, then update Step 5c to use it.

**What the transformed body must contain (from Rust snapshot analysis):**

1. **Nested $() call sites rewritten to QRL variable references.** For example, `useBrowserVisibleTask$(() => { ... })` becomes `useBrowserVisibleTaskQrl(q_Child_component_useBrowserVisibleTask_...)`. This is the same call-site rewriting logic already used for top-level extractions but applied within the body text.

2. **$-suffixed callee names renamed to Qrl-suffixed.** `useStyles$('str')` becomes `useStylesQrl(q_...)`. The callee name itself changes, AND the argument is replaced with the nested QRL variable reference.

3. **_captures[N] injection.** When the extraction has `captureNames`, the body gets `const varName = _captures[N];` declarations at the top. This is the same logic as `injectCapturesUnpacking()` in `segment-codegen.ts`.

4. **JSX event attribute rewriting.** `onClick$={() => console.log(x)}` in the body becomes `q-e:click={q_Child_component_div_q_e_click_...}`. Already handled by nested call site rewriting for JSX attrs.

**Implementation of `transformSCallBody()`:**

```typescript
function transformSCallBody(
  ext: ExtractionResult,
  allExtractions: ExtractionResult[],
  qrlVarNames: Map<string, string>,
): { transformedBody: string; additionalImports: Map<string, string> } {
  let body = ext.bodyText;
  const additionalImports = new Map<string, string>();

  // 1. Find nested extractions (children of this extraction)
  const nested = allExtractions.filter(e => e.parent === ext.symbolName);

  // 2. Rewrite nested call sites in descending position order
  //    (same approach as segment-codegen.ts nestedCallSites)
  const bodyOffset = ext.argStart;
  const sortedNested = [...nested].sort((a, b) => b.callStart - a.callStart);

  for (const child of sortedNested) {
    const childVarName = qrlVarNames.get(child.symbolName) ?? `q_${child.symbolName}`;

    // The callee name changes too: useStyles$ -> useStylesQrl
    // And the entire call (callee + arguments) is replaced

    if (child.isJsxAttr) {
      // JSX $-suffixed attribute -- not a function call, it's a JSX attribute
      // The attribute name + value gets replaced
      // This is already handled by JSX transform in the parent
      // For inline bodies, we need to handle this differently
      // Actually, for inline/hoist, JSX stays as-is in .s() bodies (transpileJsx may be false)
      // Need to check each snapshot
    }

    // For regular marker calls ($(), useTask$(), etc.):
    // Replace the entire call expression with the Qrl-suffixed callee + QRL var reference
    const relCallStart = child.callStart - bodyOffset;
    const relCallEnd = child.callEnd - bodyOffset;

    if (relCallStart >= 0 && relCallEnd <= body.length) {
      if (child.isBare) {
        // Bare $() -> just the QRL variable name
        body = body.slice(0, relCallStart) + childVarName + body.slice(relCallEnd);
      } else {
        // Named marker: callee$(args) -> calleeQrl(qrlVar) or calleeQrl(qrlVar.w([captures]))
        let replacement = child.qrlCallee + '(' + childVarName;

        // Add .w([captures]) if the child has captures
        if (child.captureNames.length > 0) {
          replacement += '.w([\n        ' + child.captureNames.join(',\n        ') + '\n    ])';
        }

        replacement += ')';
        body = body.slice(0, relCallStart) + replacement + body.slice(relCallEnd);

        // Track that we need the Qrl-suffixed callee import
        additionalImports.set(child.qrlCallee, getQrlImportSource(child.qrlCallee));
      }
    }
  }

  // 3. Inject _captures unpacking if this extraction has captures
  //    (for the extraction's own captures, not its children's)
  if (ext.captureNames.length > 0) {
    // Use the same injectCapturesUnpacking logic from segment-codegen.ts
    // Add `const varName = _captures[N];` at the top of the body
    body = injectCapturesUnpacking(body, ext.captureNames);
    additionalImports.set('_captures', '@qwik.dev/core');
  }

  return { transformedBody: body, additionalImports };
}
```
