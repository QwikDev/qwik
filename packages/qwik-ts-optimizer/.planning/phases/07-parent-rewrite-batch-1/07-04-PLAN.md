  <action>
Extend `transformSCallBody()` in rewrite-parent.ts to apply JSX transformation to the body text AFTER nested call rewriting and capture injection.

**Architecture:**
The body text at this point has already had:
1. Nested `$()` calls rewritten to QRL variable references (from existing transformSCallBody)
2. `$`-suffixed callee names renamed to Qrl-suffixed
3. `.w([captures])` appended for children with captures
4. `_captures[N]` unpacking injected

Now add step 5: JSX transpilation. The approach:

1. Take the current `body` text (after steps 1-4)
2. Wrap it as a parseable module: `const __body__ = ${body}` (the body is an arrow function expression)
3. Parse with oxc-parser (parseSync)
4. Create a MagicString from the wrapped text
5. Run transformAllJsx on the MagicString and parsed AST
6. Run signal analysis on JSX elements within the body (SignalHoister for _wrapProp, _fnSignal, _hf* declarations)
7. Extract the transformed body from the MagicString (strip the `const __body__ = ` wrapper)
8. Collect hoisted declarations (_hf0, _hf0_str, etc.) and additional imports (_jsxSorted, _wrapProp, _fnSignal, etc.)

**Gate condition:** Only apply JSX transpilation if the caller passes a flag indicating JSX should be transpiled (derived from `jsxOptions?.enableJsx` in the main function). Add a parameter to `transformSCallBody`.

**Important implementation details:**
- The transformAllJsx function takes `(source, s, program, importedNames, skipRanges, devOptions?)`. Pass an empty skipRanges array since there are no extraction regions within the body itself (nested calls already rewritten to QRL vars).
- The SignalHoister for inline bodies should produce declarations that go into the PARENT module's preamble (between the `//` separators), not into the body itself. Return hoisted declarations separately.
- Additional imports needed by JSX transform (_jsxSorted, _wrapProp, _fnSignal, Fragment as _Fragment) should be merged into the additionalImports map.
- The body text wrapping approach: `const __body__ = ` prefix, then strip after transform. This is needed because oxc-parser requires valid module-level code.

**Updating the preamble assembly (Step 6):**
Hoisted signal declarations from ALL inline .s() body transforms should be collected and added to the preamble in the same location as the existing jsxResult.hoistedDeclarations. Merge them together.

This is the simplest inline+JSX snapshot. If it passes, the core mechanism works. Then verify the others.

**NOTE about example_props_optimization:** This snapshot also needs destructuring optimization (_rawProps pattern) and _jsxSplit support. It may not pass with just JSX transpilation. That's acceptable -- focus on getting the 4 "just needs JSX" snapshots passing first.
