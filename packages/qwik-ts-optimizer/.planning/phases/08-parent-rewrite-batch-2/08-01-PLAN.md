Two changes to rewrite-parent.ts:

**Change 1: TS Stripping as Final Step**

The `rewriteParentModule` function already imports `oxcTransformSync` (line 22). After `s.toString()` produces the final parent code, add a TS stripping step when the caller indicates `transpileTs: true`.

Add a new parameter `transpileTs?: boolean` to the `rewriteParentModule` function signature. At the very end of the function, after `let finalCode = s.toString();` (or wherever the final string is assembled), add:

```typescript
if (transpileTs) {
  const stripped = oxcTransformSync('output.tsx', finalCode, { typescript: { onlyRemoveTypeImports: false } });
  finalCode = stripped.code;
}
```

This must be the LAST transformation -- after all magic-string operations are complete. This matches the proven pattern from Phase 7 Plan 05 where oxc-transform strips TS from hoist body text.

**Change 2: Capture Suppression for _auto_ Migrated Variables**

In Step 4b where `.w([captures])` is assembled, build a Set of migrated variable names from the `migrationDecisions` parameter:

```typescript
const migratedNames = new Set(
  (migrationDecisions ?? [])
    .filter(d => d.action === 'migrate')
    .map(d => d.name)
);
```

When building capture arrays for `.w()` calls, filter out any capture name that exists in `migratedNames`:

```typescript
const effectiveCaptures = ext.captureNames.filter(name => !migratedNames.has(name));
if (effectiveCaptures.length === 0) continue; // skip .w() entirely
```

This ensures that variables already exposed via `export { state as _auto_state }` don't get redundant `.w([state])` wrapping, matching Rust optimizer behavior.
