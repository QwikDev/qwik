  <action>
When a component's arrow function has destructured parameters like `({ field1, field2 })`, and any of those fields are used in signal-reactive positions (as props/children that get _wrapProp/_fnSignal treatment), the Rust optimizer transforms:

1. **Parameter**: `({ field1, field2 })` becomes `(_rawProps)` 
2. **Signal-reactive references**: `field1` in a reactive position becomes `_wrapProp(_rawProps, "field1")`
3. **Non-reactive references**: `field1` in non-reactive positions stays as `_rawProps.field1`

This is a complete parameter rewrite that must happen as a coordinated transform.

**Implementation approach:**

1. Detect component extractions whose body parameter is a destructuring pattern
2. Determine which destructured fields are used in signal-reactive positions (via analyzeSignalExpression results)
3. If any field is signal-reactive, apply the _rawProps transformation:
   - Rewrite the parameter pattern to `_rawProps`
   - Replace signal-reactive field references with `_wrapProp(_rawProps, "field")` 
   - Replace non-reactive field references with `_rawProps.field`

**Important**: Study the snapshot pairs carefully to determine the exact trigger rules. The _rawProps transform may only apply when signal wrapping is needed for at least one field. If no fields need signal wrapping, the destructuring should be preserved as-is.

This transform happens during body text generation (in the .s() body or hoist-to-const body), not during parent module rewriting.
