  <action>
Study the two snapshot expected outputs to understand the exact _regSymbol pattern:

**Expected patterns from research:**
- Import: `import { _regSymbol } from "@qwik.dev/core";`
- Import: `import { serverQrl } from "@qwik.dev/core";` (instead of regular Qrl import)
- Body wrapping: `.s(/*#__PURE__*/ _regSymbol(() => body, "hash"))` for server-tagged extractions
- Call site: `serverQrl(q_varName)` instead of `componentQrl(q_varName)` or similar

**Implementation steps:**

1. **Thread regCtxName through pipeline**: Pass `regCtxName` from TransformModulesOptions through transform.ts to rewriteParentModule (add to InlineStrategyOptions or as a separate parameter).

2. **Detect regCtxName-matched extractions**: For each extraction, check if its callee name (e.g., `server$`) matches any entry in `regCtxName` array (e.g., `['server']`). Match rule: callee name starts with or contains the regCtxName value followed by `$`.

3. **Modify body wrapping**: For matched extractions, wrap the body in `_regSymbol(() => body, "hash")` where hash is the extraction's hash. The `/*#__PURE__*/` annotation should precede the `_regSymbol` call.

4. **Modify call site generation**: For matched extractions, use `serverQrl(q_varName)` instead of the normal Qrl callee variant. This may require a new function in rewrite-calls.ts or a condition in the existing `getQrlCalleeName()` or `buildQrlDeclaration()`.

5. **Add imports**: Add `_regSymbol` and `serverQrl` to the Qwik core import line when regCtxName-matched extractions exist.

**IMPORTANT**: Study the snapshot expected output before coding. The exact format of _regSymbol and serverQrl may differ from the research assumptions.
