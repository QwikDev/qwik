2. In rewrite-parent.ts, handle inlinedQrl call site rewriting:
   The key difference from regular markers: `inlinedQrl(body, name, [captures])` must be replaced with `q_symbolName.w([captures])` (the `.w()` call preserves the original capture array INCLUDING non-identifier values like `true`).
   
   For should_preserve_non_ident_explicit_captures:
   - Input: `inlinedQrl(() => { ... }, 'task', [left, true, right])`
   - Output: `q_task.w([\n    left,\n    true,\n    right\n])`
   - The entire inlinedQrl call (from `inlinedQrl(` to the final `)`) is replaced
   
   For example_qwik_react:
   - Input: `componentQrl(inlinedQrl((props) => { ... }, "name", [reactCmpQrl]), { tagName: 'qwik-wrap' })`
   - Output: `componentQrl(q_name.w([\n    reactCmpQrl\n]), { tagName: 'qwik-wrap' })`
   - Only the inlinedQrl(...) part is replaced, the outer componentQrl call stays
   
   Implementation approach:
   - For inlinedQrl extractions, the callStart/callEnd span the entire `inlinedQrl(...)` call
   - Replace that span with `q_{symbolName}` if no captures, or `q_{symbolName}.w([captures])` if captures exist
   - The .w() array should contain the ORIGINAL capture array text (from explicitCaptures), properly formatted
   - Actually, better: format the .w() call with the capture items from the original array, preserving non-identifier values

3. QRL declaration for inlinedQrl:
   - inlinedQrl extractions still need a `const q_symbolName = qrl(...)` or `qrlDEV(...)` declaration
   - For should_preserve_non_ident_explicit_captures (dev mode, default options): uses qrlDEV with location info
   - For example_qwik_react (segment strategy, non-dev): uses regular qrl()
   - The existing QRL declaration building should work IF the extraction has correct symbolName and canonicalFilename

4. Import handling for inlinedQrl:
   - The `inlinedQrl` import itself should be REMOVED (it's replaced by qrl/qrlDEV)
   - The `_captures` import should be REMOVED from the parent (it's only used in segment bodies)
   - Add `qrl` (or `qrlDEV`) import from '@qwik.dev/core' for the QRL declaration
   - Existing import cleanup should handle removing unused `inlinedQrl` and `_captures`

5. Segment codegen for inlinedQrl:
   - The segment body is the first argument to inlinedQrl (the arrow/function expression)
   - The segment uses `_captures` references as-is (they reference the capture array by index)
