<objective>
Implement duplicate display name disambiguation in the extraction pipeline.

Purpose: The Rust optimizer appends `_1`, `_2` counters when multiple segments share the same display name (context portion). The TS optimizer currently produces duplicates, causing 13 snapshot failures. This is the dominant root cause in Phase 10.

Output: A `disambiguateExtractions()` function called at the end of `extractSegments()` that detects duplicate context portions and appends `_N` suffixes, recomputing hashes for each disambiguated name.
</objective>

  <action>
    1. Import `qwikHash` from `../hashing/siphash.js` at top of extract.ts.

    2. Add a `disambiguateExtractions()` function AFTER the `extractSegments()` function. Signature:
       ```typescript
       function disambiguateExtractions(
         extractions: ExtractionResult[],
         relPath: string,
         scope?: string,
       ): void {
       ```

    3. Logic (matches Rust `register_context_name` exactly):
       - Compute `fileStem` prefix: take basename of relPath (after last `/`), this is the `prefix = basename + '_'`.
       - Create `nameCounters = new Map<string, number>()`.
       - Iterate `extractions` in order:
         a. Extract `contextPortion`: if `ext.displayName.startsWith(prefix)`, slice off prefix; else use full displayName.
         b. Look up `contextPortion` in `nameCounters`:
            - If NOT present: set `nameCounters.set(contextPortion, 0)` -- first occurrence, no suffix.
            - If present: increment: `const newIndex = existing + 1; nameCounters.set(contextPortion, newIndex);`
              Then update the extraction:
              ```
              const newContext = contextPortion + '_' + newIndex;
              const newHash = qwikHash(scope, relPath, newContext);
              ext.displayName = prefix + newContext;  // where prefix = basename + '_'
              ext.hash = newHash;
              ext.symbolName = newContext + '_' + newHash;
              ext.canonicalFilename = ext.displayName + '_' + newHash;
              ```

    4. Call `disambiguateExtractions(results, relPath, scope)` at the end of `extractSegments()`, just before `return results;` (line ~735).

    IMPORTANT: The counter is per-file (per extractSegments call), matching Rust's per-QwikTransform scope. The first occurrence gets index 0 (no suffix), the second gets `_1`, third `_2`, etc.
  </action>
