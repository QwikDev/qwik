  <action>
    1. Add a `let markerCallDepth = 0;` variable at the top of the `walk()` callback scope in `extractSegments()`, alongside the existing `results` array and `pushedNodes` map.

    2. In the `enter()` callback, AFTER detecting a marker call (where `isMarkerCall()` returns true and extraction is performed), increment `markerCallDepth++`. This should happen when entering the CallExpression node that is a marker call.

    3. In the `leave()` callback, detect when leaving a marker call's CallExpression and decrement `markerCallDepth--`. To track which CallExpression nodes are marker calls, add them to a Set during enter:
       ```typescript
       const markerCallNodes = new Set<any>();
       ```
       In enter, when isMarkerCall: `markerCallNodes.add(node);`
       In leave: `if (markerCallNodes.has(node)) { markerCallDepth--; markerCallNodes.delete(node); }`

    4. Gate the JSX `$-suffixed` attribute extraction (lines 625-720) on `markerCallDepth > 0`:
       Change the condition at line 645 from:
       ```typescript
       if (jsxAttrName !== null) {
       ```
       to:
       ```typescript
       if (jsxAttrName !== null && markerCallDepth > 0) {
       ```

    This ensures JSX attrs like `onClick$={() => ...}` are only extracted when they appear inside the argument of a `component$()`, `qwikify$()`, or other marker call. Plain arrow functions with `$-suffixed` JSX attrs (like the `App` function in example_jsx_import_source) won't produce extractions.

    NOTE: The existing check at line 650 (`if expr is CallExpression && isMarkerCall -> skip`) still applies inside marker scope -- this prevents double-extraction of `onClick$={$(() => ...)}`.
