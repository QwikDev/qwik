(e.g., `const handleClick$ = q_xxx`) but the Rust optimizer emits them with .w() capture wiring
(e.g., `const handleClick$ = q_xxx.w([toggleSig])`).

1. In generateSegmentCode(), find where nested QRL call sites are rewritten into `const varName = q_symbolName` assignments.
2. For each nested call site that has non-empty captureNames, append `.w([captureNames.join(', ')])` to the QRL reference.
3. Use the existing nestedCallSites data which already carries captureNames from the extraction phase.
4. Debug with `npx vitest run tests/optimizer/convergence.test.ts` using specific snapshot names like ternary_prop, moves_captures_when_possible, should_extract_single_qrl to verify the .w() pattern appears.
5. Check the rewrite-parent.ts implementation of .w() for the exact format (the parent module already does this correctly -- mirror that format in segment codegen).

Key insight: The nestedCallSites array from transform.ts already knows which captures each nested extraction has. The segment codegen just needs to use that data when emitting QRL variable declarations.

Expected diff in output:
- Before: `const handleClick$ = q_Cmp_component_handleClick_WawHV3HwS1A`
- After:  `const handleClick$ = q_Cmp_component_handleClick_WawHV3HwS1A.w([toggleSig])`
