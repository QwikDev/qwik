**Change 3: Suppress _captures import for inline strategy**

In `src/optimizer/rewrite-parent.ts`, at lines 1173-1182, the `_captures` import is added for ALL strategies when any extraction has captures. For inline strategy, `_captures` should NOT be imported at module level because captures are delivered via `.w()` in the parent scope, not via `import { _captures }`.

Change lines 1175-1182 from:
```typescript
const needsCapturesImport = extractions.some(
  (e) => !e.isSync && e.captureNames.length > 0 && !(inlineOptions && isStrippedSegment(
    e.ctxName, e.ctxKind, inlineOptions.stripCtxName, inlineOptions.stripEventHandlers,
  )),
);
if (needsCapturesImport && !alreadyImported.has('_captures')) {
  neededImports.set('_captures', '@qwik.dev/core');
}
```

To:
```typescript
// _captures import needed if any non-stripped extraction has captures
// BUT NOT for inline strategy -- inline mode delivers captures via .w()
// in parent scope, not via import { _captures }
if (!isInline) {
  const needsCapturesImport = extractions.some(
    (e) => !e.isSync && e.captureNames.length > 0 && !(inlineOptions && isStrippedSegment(
      e.ctxName, e.ctxKind, inlineOptions.stripCtxName, inlineOptions.stripEventHandlers,
    )),
  );
  if (needsCapturesImport && !alreadyImported.has('_captures')) {
    neededImports.set('_captures', '@qwik.dev/core');
  }
}
```

Also, inside `transformSCallBody` at line 660, suppress the `_captures` additionalImport when in inline mode. The simplest approach: the `additionalImports.set('_captures', ...)` at line 660 is fine to keep -- it gets filtered by the caller. But actually, the caller at line 1477-1481 adds ALL additionalImports without checking. So we need to filter it there too.

In the `processExtraction` function at lines 1477-1481, add a filter:
```typescript
for (const [sym, src] of additionalImports) {
  // Skip _captures import for inline strategy -- captures delivered via .w()
  if (sym === '_captures' && isInline) continue;
  if (!alreadyImported.has(sym) && !neededImports.has(sym)) {
    neededImports.set(sym, src);
  }
}
```
