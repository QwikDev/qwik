**Part A: Fix computeFlags to match SWC bitmask order**

Replace the `computeFlags` function (around lines 301-327) with SWC-aligned semantics:

```typescript
export function computeFlags(
  hasVarProps: boolean,
  childrenType: 'none' | 'static' | 'dynamic',
  inLoop: boolean = false,
): number {
  let flags = 0;
  // Bit 0 (value 1): static_listeners -- ALL props are const (no var props)
  // Source: swc-reference-only/transform.rs lines 2644-2653
  if (!hasVarProps) {
    flags |= 1;
  }
  // Bit 1 (value 2): static_subtree -- children are static/none
  if (childrenType !== 'dynamic') {
    flags |= 2;
  }
  // Bit 2 (value 4): moved_captures -- captures moved via q:p/q:ps loop context
  if (inLoop) {
    flags |= 4;
  }
  return flags;
}
```

Update the JSDoc comment above `computeFlags` to reflect the corrected semantics:
- Bit 0 (1): static_listeners -- set when no var props at all
- Bit 1 (2): static_subtree -- set when children are static or none
- Bit 2 (4): moved_captures -- set when in loop context

Remove any stale comments about "always set outside loop" or "bit 0 = children".

However, the simpler fix is: _jsxSorted calls are themselves function calls, so ALL nested JSX elements (already transformed) are function calls. But SWC treats `_jsxSorted`/`_jsxC` as immutable (they are in `jsx_functions` list). Keep the current behavior for nested JSX elements -- the regex check is adequate because _jsxSorted with varProps implies reactivity.

**Part B: Fix component vs HTML element prop routing in processProps**

In the classifyProp fallback section of `processProps` (around lines 810-831), the current code routes props based on `classification === 'var'` without considering whether the tag is an HTML element or component. SWC has opposite defaults:

- HTML elements: props go to constEntries by default, only var props go to varEntries
- Component elements: props go to varEntries by default, only explicitly const props go to constEntries

The current routing (lines 822-831) is:
```typescript
if (beforeSpread) {
  beforeSpreadEntries.push(entry);
} else if (classification === 'var') {
  varEntries.push(entry);
} else {
  constEntries.push(entry);
}
```

Change to account for component vs HTML element routing:
```typescript
if (beforeSpread) {
  beforeSpreadEntries.push(entry);
} else if (tagIsHtml) {
  // HTML elements: const by default, only var if classifyProp says var
  if (classification === 'var') {
    varEntries.push(entry);
  } else {
    constEntries.push(entry);
  }
} else {
  // Component elements: var by default, only const if classifyProp says const
  // Source: swc-reference-only/transform.rs lines 3328-3344
  if (classification === 'const') {
    constEntries.push(entry);
  } else {
    varEntries.push(entry);
  }
}
```
