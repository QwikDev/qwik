  <action>
In `classifyProp` (jsx-transform.ts), the `CallExpression` case currently returns `'var'` unconditionally. SWC's `is_const_expr` has a `const_idents` set that includes synthetic Qwik functions like `_qrlSync`, `_wrapProp`, `_wrapSignal`, and `_fnSignal`. These are always const-valued because they return stable QRL/signal wrapper objects.

Add a special case BEFORE the generic `CallExpression` return:

```typescript
case 'CallExpression': {
  // SWC const_idents: certain Qwik internal calls are known to produce const values
  const callee = exprNode.callee;
  if (callee?.type === 'Identifier') {
    const CONST_CALL_IDENTS = new Set([
      '_qrlSync', '_wrapProp', '_wrapSignal', '_fnSignal',
      'qrl', 'inlinedQrl', '_noopQrl',
    ]);
    if (CONST_CALL_IDENTS.has(callee.name)) {
      return 'const';
    }
  }
  return 'var';
}
```

This ensures:
- `_qrlSync(fn, "str")` event handlers land in constProps (3rd arg of _jsxSorted) 
- `_wrapProp(signal)` expressions are classified as const
- Flags bitmask bit 0 (static_listeners) is set correctly when all props are const calls

This is per research assumption A1: "_qrlSync is a known Qwik internal, safe to hardcode as const." The full const_idents tracking (for arbitrary user-declared const bindings) remains deferred.
